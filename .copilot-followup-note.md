# Followup note for PR #190

The original problem statement speculated about a `"premium"` tier. That is incorrect — there is no `"premium"` tier in this product. The only paid tier is `"pro"`, billed either monthly or yearly (the cadence is handled separately at checkout, not via this prop).

## Required adjustment to `RequireSubscription` typing

In `frontend/src/App.tsx`, type the `tier` prop as a plain string (or the literal `"pro"`), NOT as `"pro" | "premium"`:

```tsx
function RequireSubscription({
  tier = "pro",
  intent,
  children,
}: {
  tier?: string;     // <-- use string, not "pro" | "premium"
  intent: string;
  children: React.ReactNode;
}) {
  ...
}
```

Everything else in the original problem statement still applies. After making this adjustment, delete this `.copilot-followup-note.md` file before opening/finalizing the PR.
