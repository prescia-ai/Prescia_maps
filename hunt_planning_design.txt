TECH STACK
----------
Frontend:
  - Framework:       React 18 + TypeScript, bundled by Vite 5
  - Routing:         react-router-dom v7  (frontend/src/App.tsx)
  - Mapping:         Leaflet 1.9 + react-leaflet 4, leaflet.heat for heatmap
  - Server state:    TanStack Query (@tanstack/react-query v5)
  - HTTP client:     axios v1.7 with shared api instance (frontend/src/api/client.ts)
  - Auth:            Supabase (@supabase/supabase-js v2) — JWT verified by backend
  - Styling:         Tailwind CSS v3 + PostCSS

Backend:
  - Framework:       FastAPI 0.111, served by Uvicorn  (backend/main.py)
  - ORM:             SQLAlchemy 2 async  (backend/app/models/database.py)
  - Database:        PostgreSQL 16 + PostGIS 3.4 (docker-compose.yml)
  - Migrations:      Alembic  (backend/alembic/)
  - Auth:            Supabase JWT verified with python-jose
  - Geocoding:       geopy, httpx, wiki_geocoding.py
  - Land access:     PAD-US PMTiles + REST API  (backend/app/services/land_access.py)
  - Image storage:   Google Drive via OAuth2  (backend/app/api/google_auth.py)
  - AI/Scoring:      optional sentence-transformers for semantic hotspot scoring

Hosting / Infra:
  - Docker Compose for local dev
  - Supabase for auth (JWT only — no Supabase DB)
  - Google Drive for user-uploaded images

4.1 DATA MODEL
--------------

hunt_plans table (add to backend/app/models/database.py):

  class HuntPlanStatus(str, enum.Enum):
      draft      = "draft"
      scheduled  = "scheduled"
      completed  = "completed"
      cancelled  = "cancelled"

  class HuntPlan(Base):
      __tablename__ = "hunt_plans"

      id               UUID PK default uuid4
      owner_id         UUID NOT NULL INDEX          -- FK → users.id
      group_id         UUID NULLABLE INDEX          -- optional group association

      title            String(200) NOT NULL
      description      Text NULLABLE

      latitude         Float NOT NULL               -- primary hunting area
      longitude        Float NOT NULL
      geom             Geometry(POINT, 4326)        -- spatial index
      area_polygon     JSONB NULLABLE               -- GeoJSON polygon

      planned_date     DateTime(timezone=True) NOT NULL
      planned_end      DateTime(timezone=True) NULLABLE

      target_species   JSONB NULLABLE               -- ["whitetail", "turkey"]
      privacy          String(20) default="private" -- private | party | group
      status           Enum(HuntPlanStatus) NOT NULL default=draft

      weather_snapshot JSONB NULLABLE               -- temp, wind, precip, condition
      moon_phase       String(50) NULLABLE          -- "waxing gibbous"
      sunrise_time     String(10) NULLABLE          -- "06:34"
      sunset_time      String(10) NULLABLE          -- "19:12"

      notes            Text NULLABLE
      gear_checklist   JSONB NULLABLE               -- [{item, checked}]

      converted_pin_id UUID NULLABLE               -- FK → user_pins.id (after conversion)

      created_at       DateTime server_default now()
      updated_at       DateTime server_default now() onupdate now()


hunt_plan_members table:

  plan_id    UUID PK   -- FK → hunt_plans.id
  user_id    UUID PK   -- FK → users.id
  role       String(20) default="member"   -- owner | member
  status     String(20) default="invited"  -- invited | accepted | declined
  added_at   DateTime server_default now()


hunt_plan_stands table:

  id           UUID PK default uuid4
  plan_id      UUID NOT NULL INDEX          -- FK → hunt_plans.id
  label        String(100) NOT NULL         -- "North Stand", "Blind A"
  latitude     Float NOT NULL
  longitude    Float NOT NULL
  assigned_to  UUID NULLABLE                -- FK → users.id
  stand_type   String(50) NULLABLE          -- treestand | ground_blind | drop_zone
  notes        Text NULLABLE

4.2 API ENDPOINTS
-----------------
New file backend/app/api/hunt_plans.py, registered in main.py.
All endpoints under /api/v1/hunt-plans.

  POST   /hunt-plans                              Create a plan
  GET    /hunt-plans/me                           List my plans (owner or party member)
  GET    /hunt-plans/{plan_id}                    Get single plan
  PUT    /hunt-plans/{plan_id}                    Update plan (owner only)
  DELETE /hunt-plans/{plan_id}                    Delete plan (owner only)
  PATCH  /hunt-plans/{plan_id}/status             Transition status

  POST   /hunt-plans/{plan_id}/members            Invite by username
  DELETE /hunt-plans/{plan_id}/members/{uid}      Remove from party
  PATCH  /hunt-plans/{plan_id}/members/me         Accept / decline invitation

  POST   /hunt-plans/{plan_id}/stands             Add stand
  PUT    /hunt-plans/{plan_id}/stands/{sid}       Update stand
  DELETE /hunt-plans/{plan_id}/stands/{sid}       Delete stand

  GET    /hunt-plans/{plan_id}/weather            Fetch / refresh weather snapshot

  POST   /hunt-plans/{plan_id}/convert            Convert plan → UserPin logged hunt

  GET    /hunt-plans/map-pins                     Upcoming plan pins for map layer

convert endpoint logic:
  1. Validate caller is plan owner
  2. Create UserPin from plan.latitude, longitude, planned_date, title, notes
  3. Set hunt_plan.converted_pin_id = new_pin.id, status = "completed"
  4. Trigger check_all_badges() (same call used in pins.py)
  5. Return the new UserPin

4.5 OPTIONAL INTEGRATIONS
--------------------------
Weather Forecast:
  - Backend proxies to Open-Meteo (free, no API key) or WeatherAPI
  - Add WEATHER_API_KEY setting to backend/app/config.py
  - Response stored as JSONB in hunt_plans.weather_snapshot
  - Frontend "Refresh forecast" button calls GET /hunt-plans/{id}/weather

Moon Phase:
  - Pure Python calculation or 'ephem' library
  - Computed from planned_date; stored in hunt_plans.moon_phase
  - Displayed as icon/emoji in plan header

Sunrise / Sunset:
  - Free Sunrise-Sunset API or Python 'astral' library
  - Computed from lat/lon + date; stored in sunrise_time / sunset_time

Historical Sightings Overlay:
  - Fetch authenticated user's past UserPin records near plan area
  - Overlay as semi-transparent markers in PlanDetailPage map

Notifications / Reminders:
  - Reuse existing Notification model and create_notification() directly
  - New notification type: "hunt_plan_reminder"
  - Background task (APScheduler) queries plans with planned_date in next 24h
    and status=scheduled, then calls create_notification() for owner + party
  - NotificationBell component already handles all types — no UI changes needed

Sharing & Collaboration:
  - Party invitations trigger "hunt_plan_invite" notifications
  - Accepted members can view plan detail (privacy check: user in party_members)
  - Group-scoped plans visible to all active group members (plan.group_id pattern
    mirrors Post.group_id used in feed.py)

Plan → Logged Hunt Conversion (see 4.2 convert endpoint above):
  - Frontend CTA in PlanDetailPage
  - On success: navigate to MapPage with new pin highlighted
  - Show BadgeDisplay popup (reuse existing BadgeDisplay.tsx)
