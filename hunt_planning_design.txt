TECH STACK
----------
Frontend:
  - Framework:       React 18 + TypeScript, bundled by Vite 5
  - Routing:         react-router-dom v7  (frontend/src/App.tsx)
  - Mapping:         Leaflet 1.9 + react-leaflet 4, leaflet.heat for heatmap
  - Server state:    TanStack Query (@tanstack/react-query v5)
  - HTTP client:     axios v1.7 with shared api instance (frontend/src/api/client.ts)
  - Auth:            Supabase (@supabase/supabase-js v2) — JWT verified by backend
  - Styling:         Tailwind CSS v3 + PostCSS

Backend:
  - Framework:       FastAPI 0.111, served by Uvicorn  (backend/main.py)
  - ORM:             SQLAlchemy 2 async  (backend/app/models/database.py)
  - Database:        PostgreSQL 16 + PostGIS 3.4 (docker-compose.yml)
  - Migrations:      Alembic  (backend/alembic/)
  - Auth:            Supabase JWT verified with python-jose
  - Geocoding:       geopy, httpx, wiki_geocoding.py
  - Land access:     PAD-US PMTiles + REST API  (backend/app/services/land_access.py)
  - Image storage:   Google Drive via OAuth2  (backend/app/api/google_auth.py)
  - AI/Scoring:      optional sentence-transformers for semantic hotspot scoring

Hosting / Infra:
  - Docker Compose for local dev
  - Supabase for auth (JWT only — no Supabase DB)
  - Google Drive for user-uploaded images

4.1 DATA MODEL
--------------
Single new table. NO auxiliary tables in v1.

hunt_plans (add to backend/app/models/database.py):

  class HuntPlanStatus(str, enum.Enum):
      idea      = "idea"
      planned   = "planned"
      done      = "done"
      archived  = "archived"

  class HuntPlan(Base):
      __tablename__ = "hunt_plans"

      id               UUID PK default uuid4
      owner_id         UUID NOT NULL INDEX       -- FK → users.id  (indexed)

      title            String(200) NOT NULL
      planned_date     DateTime nullable
      site_type        String(20)                -- dirt|beach|water|park|yard|club_hunt
      status           Enum(HuntPlanStatus)      -- idea|planned|done|archived

      notes            Text nullable
      geom             Geometry(POINT, 4326)     -- centroid, auto-derived from area_geojson
      area_geojson     JSONB NOT NULL            -- full polygon/rectangle/circle GeoJSON

      in_zone_markers  JSONB nullable            -- [{id, lat, lng, type, label, note}]
                                                 --   type: dig_target|avoid|access|
                                                 --         already_detected

      gear_checklist   JSONB nullable            -- [{item, checked}]
      permission       JSONB nullable            -- {owner_name, contact, status,
                                                 --   expiry, notes}  (text fields only;
                                                 --   no automation, no PDF generation)
      view_snapshot    JSONB nullable            -- {center: [lat,lng], zoom,
                                                 --   layers: {...}}  (restores map view
                                                 --   when plan is reopened)
      photo_urls       JSONB nullable            -- [str]  Google Drive URLs
                                                 --   (reuse existing upload pattern)

      created_at       DateTime server_default now()
      updated_at       DateTime server_default now() onupdate now()

  Columns explicitly NOT present in v1:
    group_id, weather_snapshot, moon_phase, sunrise_time, sunset_time,
    converted_pin_id, privacy, hunt_plan_members FK, hunt_plan_stands FK.

4.2 API ENDPOINTS
-----------------
New file backend/app/api/hunt_plans.py, registered in main.py.
All endpoints under /api/v1/hunt-plans.

  POST   /hunt-plans
    Create a plan. Body: HuntPlanCreate. Centroid geom auto-derived from
    area_geojson on the backend. Returns HuntPlanResponse.

  GET    /hunt-plans/me
    List the caller's plans. Query params: search (string), sort (created_at |
    planned_date | title), order (asc | desc), site_type filter, status filter.
    Returns HuntPlanListResponse. Archived plans are included when
    status=archived is passed; excluded otherwise by default.

  GET    /hunt-plans/{plan_id}
    Get a single plan (owner only). Returns HuntPlanResponse.

  PUT    /hunt-plans/{plan_id}
    Full update (owner only). Returns HuntPlanResponse.

  DELETE /hunt-plans/{plan_id}
    Delete plan (owner only). 204 No Content.

  PATCH  /hunt-plans/{plan_id}/status
    Transition status: idea → planned → done → archived (any direction).
    Body: { status: HuntPlanStatus }. Returns HuntPlanResponse.

  POST   /hunt-plans/{plan_id}/duplicate
    Clone an existing plan. Copies all fields except id, created_at, updated_at.
    Status resets to "idea". Returns the new HuntPlanResponse.

  GET    /hunt-plans/map-pins
    Lightweight list for the Hunt Plans map layer. Returns
    [{ id, title, lat, lng, status, site_type }] for the caller's plans.
    Excludes archived plans by default; pass ?include_archived=true to include.

  GET    /hunt-plans/{plan_id}/export.gpx
    GPX export of the plan zone polygon + in-zone markers as waypoints.

  GET    /hunt-plans/{plan_id}/export.pdf
    Printable PDF summary of the plan (title, date, site type, notes,
    gear checklist, permission info, map thumbnail if feasible).

4.7 EXPLICITLY DEFERRED BEYOND v1
----------------------------------
The following items were discussed and deliberately excluded from v1.
They are listed here so future contributors know these were intentional
decisions, not oversights.

  hunt_plan_members table / party invites
    Fine-grained per-user sharing. Deferred because plans are private in v1.
    Revisit if users request collaborative planning.

  hunt_plan_stands table
    Replaced in v1 by JSONB in_zone_markers on the plan record.
    A separate relational table (with assigned_to FK, stand_type enum, etc.)
    can be introduced later if marker complexity grows.

  Weather snapshot, weather service integration
    weather_snapshot field, Open-Meteo / WeatherAPI proxy, WEATHER_API_KEY
    config setting, PlanWeatherWidget component. Deferred.

  Moon phase, sunrise/sunset times
    moon_phase, sunrise_time, sunset_time fields and ephem/astral libraries.
    Deferred.

  Plan → UserPin conversion endpoint ("Convert to Hunt Log")
    POST /hunt-plans/{id}/convert, converted_pin_id field, the frontend
    "Convert to Hunt Log" button, and BadgeDisplay trigger. Live hunt mode
    (GPS breadcrumbs, real-time find logging) is its own future feature.
    Deferred.

  Notification / reminder system for plans
    hunt_plan_reminder notification type, APScheduler background tasks,
    NotificationBell changes for plan types. Deferred.

  Calendar view component
    PlanCalendarView month-grid. Deferred.

  Group sharing / group_id on plans
    group_id field, group-scoped plan visibility, Post.group_id pattern
    applied to plans. Deferred.

  AI scoring of plans
    Extending the existing 0–100 engine to grade a plan's site density,
    land-access status, and historical aerial signals. Deferred.

  Live hunt mode
    "Start Hunt" button, GPS breadcrumb trail, real-time find logging,
    auto-conversion to hunt log on end. A separate future feature.

  Multi-step wizard
    The original 7-step create wizard (Location → Date → Details → Party →
    Stands → Gear → Review). v1 uses a single-page create flow instead.
